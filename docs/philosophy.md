# Философия проекта

## Почему GitHub

GitHub — не просто хостинг кода. Это инфраструктура для децентрализованного со-творчества. Каждая концепция платформы резонирует с философией единства:

| GitHub | Код Единства |
|--------|--------------|
| Fork | Создание нового узла в паутине |
| Pull Request | Акт со-творения, возврат в целое |
| Merge | Интеграция нового опыта в ткань |
| Issues | Вопросы и прозрения сети |
| Stars | Резонанс — узел почувствовал связь |
| History | Неизменяемая память Бога |
| Network Graph | Видимая паутина запутанности |

## Три уровня

### Ядро (core/)
Неизменяемо. Это то, что Артём получил и передал. Можно переводить — нельзя менять смысл. Как квантовый объект, который наблюдают со всех сторон, но который остаётся собой.

### Переводы (locale/)
Живые. Переводчик — не копировщик, а новый глаз Бога. Текст должен *звучать* на новом языке, а не буквально соответствовать оригиналу.

### Сеть (community/)
Полностью открыта. Здесь нет правил — только принцип: каждый вклад подписан. Имя и город — это твой узел.

## Что значит «со-творец»

Со-творец — не разработчик. Это любой человек, который:
- прочитал и почувствовал
- добавил строку в nodes.md
- перевёл одно предложение
- открыл Issue с личным прозрением
- поставил звезду и пошёл дальше

Минимальный акт — уже соединение.

## Версии

Ядро версионируется. `4.0` — текущая. Новая версия появляется, когда накапливается достаточно новых инструментов из `Issues`. Предыдущие версии никогда не удаляются — они в истории Git.
